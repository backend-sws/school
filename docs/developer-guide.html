Architecture & Multitenancy

PDS Education utilizes a Domain-Based Context Resolution system to power a multi-institution ecosystem from a single codebase.

🔄 Request Lifecycle Sequence

sequenceDiagram
    participant U as User Browser
    participant N as Nginx/Ingress
    participant RM as ResolveInstitutionFromDomain (Middleware)
    participant IC as InstitutionContext
    participant P as Policy/Middleware
    participant C as Controller

    U->>N: GET school.lvh.me/dashboard
    N->>RM: Forward request
    RM->>RM: Extract 'school' from host
    RM->>DB: Query Institution where domain = 'school'
    DB-->>RM: Institution (ID: 10, Type: school)
    RM->>RM: Config::set('ems.default_institution_id', 10)
    RM->>RM: Config::set('ems.default_institution_type', 'school')
    RM->>P: EnsurePermission ('dashboard_view')
    P->>DB: Check roles_assigned (user_id, scope_id=10, perm)
    DB-->>P: Authorized
    P->>C: Handle Request

The ResolveInstitutionFromDomain middleware is the single point of entry. It overrides app.url and sets the ems.default_institution_id context for all downstream Eloquent queries.

Core Middleware Layers

Middleware Responsibility
ResolveInstitutionFromDomain Detects tenant context from subdomain, sets global config for scoping.
EnsurePermissionGroup Maps route groups (e.g., academic_setup) to permission sets in config/route_permissions.php.
CheckSubscription Validates if the tenant's SubscriptionTier includes the module requested.
HandleInertiaRequests Shares global context (auth, flash, institution data) to the React frontend.

Developer Best Practices

Global Scoping

Never query models without scoping. Use InstitutionContext::getActiveInstitutionId($user) to fetch the current tenant ID.

Form Pattern

All forms MUST use ControlledFormComponent with zod. This ensures consistent field layout and validation UX.

API Modules

Keep API calls out of React components. Centralize them in lib/api/<feature>Api.ts and export a CRUD object.

Error Mapping

Don't hardcode error strings. Use ApiErrorMap::respond() to pull user-facing messages from config/api_error_maps.php.
